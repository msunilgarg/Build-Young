# Build Young — cohort simulation output

All money here is **simulated**. Build Young is financial education, not licensed
financial advice. These files are generated by an offline harness that drives the
real dashboard simulation engine (the same `advance()` used in the student app),
so the numbers match what a student would see in the live simulation.

## How it was generated

- Harness: `scripts/simulateBatch.js` (`runCohort(batchId)`).
- Runner: `test/simulate.batch.test.js` (run with `npm run simulate` or `npm test`).
- The engine is imported READ-ONLY from `src/App.jsx` + `src/marketMedia.js`.
- Each student is run through the full lifecycle: 12 weekly classes (`phase:"course"`)
  then 6 monthly check-ins (`phase:"checkin"`), 18 periods total. Each period applies
  one $10,000 paycheck (split per the student's settings), then the period's shared
  market event from `marketEventFor(...)`, then that week's decisions.
- **Reproducible:** the only randomness is the hustle bonus in `advance()`. The harness
  installs a seeded PRNG (seed 12345, mixed with the batch id) and restores `Math.random`
  afterward, so re-running produces identical files.

## Funnel modeled

discover (source) → (maybe) book a free 15-min call → enroll → engage weekly →
complete the course → finish 6 check-ins → graduate. Each student's `funnel` field
records their stage progression, and `discoverySource` / `bookedCall` capture the top.

## Files

```
simulation-output/
  README.md                 ← this file
  fall-ms-mon/
    cohort-summary.json     ← roster, prize ranking, averages/min/max, risk breakdown, notable outcomes
    cohort.csv              ← one row per student for spreadsheet review
    <student-slug>.json     ← full per-student journey (15 files)
  fall-hs-wed/
    cohort-summary.json     ← roster, prize ranking, averages/min/max, risk breakdown, notable outcomes
    cohort.csv              ← one row per student for spreadsheet review
    <student-slug>.json     ← full per-student journey (15 files)
```

Each per-student file contains: `profile` (name, email, track, discovery source,
booked-call flag, persona, settings), `funnel`, `weekByWeek` (per-period market event,
human-readable decisions, net worth, cash, savings, retirement, holdings, credit score,
home/car/card/PE/insurance state), `emailsReceived` (welcome + recaps + the 3-email
pre-class market-news drips), and `final` (net worth, breakdown, credit score, narrative).

## A note on the engine's economics (a real, teachable finding)

On a $10,000/period salary re-tuned to a realistic young-adult budget, students are
LIQUIDITY-constrained. Big lump purchases are gated by available funds, exactly as the
dashboard's disabled buy buttons enforce, so the harness models the realistic behavior of
*saving toward a goal and buying when it's affordable*. In practice: a **car** ($6,000
down) is reachable for steady savers; a **home** ($20,000 down) is only reachable by
max-rate savers, and only by the final check-ins; and the **$15,000 private-equity** buy
is effectively unreachable for everyone within the 18 periods given how little cash
accumulates — so no persona clears it this run (the field is still captured, always $0).
Heavy auto-investors finish with the largest invested base but stay cash-poor — a genuine
trade-off worth showing students.
