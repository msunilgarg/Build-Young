## [ ] SEO content: resources/blog articles for long-tail organic traffic  ·  risk: low–med
**Why:** the site is essentially one page, so Google has little to rank. Focused articles each target a
real query the landing page can't, add indexable surface, and act as backlink magnets — the on-site
lever that pulls a small site up for competitive terms over time. (See `marketing/seo-checklist.md`.)
**Approach when promoted:**
- Static HTML under `public/resources/` (same crawl-proof pattern as `public/privacy.html` /
  `terms.html`) — fully readable without JS, no SPA/prerender risk. URLs like
  `/resources/hands-on-ai-for-high-schoolers.html`.
- Brand-styled (site colors/fonts, inline); add each page to `public/sitemap.xml`; link from the
  footer ("Resources"); cross-link each article to Enroll / Book-a-call.
- **Start with ONE cornerstone article** — *"Hands-on AI for high schoolers: what it actually means"*
  (the exact target phrase) — get it indexed, watch GSC → Performance for impressions, then expand.
**Candidate topics:** "Do teens need to code to build with AI?", "What a high schooler can actually
build (and ship) in 12 weeks", "Hands-on AI vs. watching AI tutorials", "How to tell if your teen is
ready to build".
**Then:** Request Indexing for the new URLs + confirm they're in the sitemap.
